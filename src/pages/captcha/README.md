封装可扩展组件思路
  1. 可读性(代码格式统一清晰，注释完整，代码结构层次分明，编程范式使用得当)
  2. 可用性(代码功能完整，在不同场景都能很好兼容，业务逻辑覆盖率)
  3. 复用性(代码可以很好的被其他业务模块复用)
  4. 可维护性(代码易于维护和扩展，并有一定的向下/向上兼容性)
  5. 高性能

滑块验证码
  用于登录注册、活动、论坛短信等高险业务场景场景的人机验证服务
  功能点梳理
    1. 控制何时显示和隐藏验证码(visible)
    2. 滑动验证码
    3. 控制滑动验证码大小(width,height)
    4. 控制滑块的样式(边长1,半径r)
    5. 可以自定义滑块提示文本的内容(text)
    6. 滑动验证码的图片自定义(imgUrls)
    7. 用户能刷新重试,并能监听到刷新动作(onRefresh)
    8. 提供用户在验证成功或者失败后的事件控制(onSuccess,onFail)
